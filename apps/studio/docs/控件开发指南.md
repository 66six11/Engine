# Studio 控件开发指南

本文定义 `apps/studio` 的 Avalonia 控件和样式策略。当前路线是：**官方 Fluent theme 提供内置控件默认模板，Studio 只做必要覆盖和项目专用控件**。不要再为 `Button`、`TextBox`、`Menu`、`ListBox`、`GridSplitter` 这类内置控件批量重写完整 `ControlTheme`，除非明确要接管它的完整视觉树、交互状态和可访问性成本。

官方资料：

- Avalonia Themes: <https://docs.avaloniaui.net/docs/styling/themes>
- Avalonia Style selectors: <https://docs.avaloniaui.net/docs/styling/style-selectors>
- Avalonia Control themes: <https://docs.avaloniaui.net/docs/basics/user-interface/styling/control-themes>
- Avalonia Control template walkthrough: <https://docs.avaloniaui.net/docs/styling/control-template-walkthrough>
- Avalonia Pseudoclasses: <https://docs.avaloniaui.net/docs/reference/styles/pseudo-classes>
- Avalonia XAML preview and design settings: <https://docs.avaloniaui.net/docs/app-development/xaml-preview-and-design-settings>
- Avalonia Controls library: <https://docs.avaloniaui.net/controls>

## 1. 主题加载

`App.axaml` 必须先加载官方基础 theme，再加载 Studio 覆盖样式：

```xml
<Application.Styles>
    <FluentTheme DensityStyle="Compact" />
    <StyleInclude Source="avares://Editor/UI/Controls/Base/Native/Index.axaml" />
    <StyleInclude Source="avares://Editor/UI/Styles/Shell/MainMenu.axaml" />
</Application.Styles>
```

原因：

- Avalonia 的内置控件大量依赖 theme 提供默认 `ControlTemplate`。
- 没有官方 theme 时，内置控件可能缺少视觉树、默认尺寸、交互状态或可命中区域。
- Studio 当前没有资源维护一整套完整控件库，所以不走完全自定义 theme。

Studio 的资源字典只放 token 和项目专用控件主题：

```xml
<Application.Resources>
    <ResourceDictionary>
        <ResourceDictionary.MergedDictionaries>
            <ResourceInclude Source="avares://Editor/UI/Styles/Tokens/DeepDarkColors.axaml" />
            <ResourceInclude Source="avares://Editor/UI/Styles/Tokens/EditorMetrics.axaml" />
            <ResourceInclude Source="avares://Editor/UI/Controls/Base/IconButton.axaml" />
        </ResourceDictionary.MergedDictionaries>
    </ResourceDictionary>
</Application.Resources>
```

## 2. 原生控件全局覆盖

全局覆盖 Avalonia 原生控件的样式必须放在 `UI/Controls/Base/Native`，并按控件或控件族拆分存放：

```text
UI/Controls/Base/Native
  Index.axaml
  Typography.axaml
  TextBox.axaml
```

规则：

- `Index.axaml` 只做聚合入口，由 `App.axaml` 引用。
- 每个拆分文件只覆盖一个原生控件或紧密控件族，例如 `Menu.axaml` 可以同时覆盖 `Menu` 和 `MenuItem`。
- 每个拆分文件必须包含 `Design.PreviewWith`，用于预览该控件族的默认、disabled、常用状态。
- 只通过公开属性覆盖字体、字号、cursor、padding、颜色、背景、边框等，不依赖 Fluent 内部模板结构。
- 不在这里写业务 class。业务 class 样式放到对应 Shell/Feature 样式文件。

字体族和默认字号属于继承式 typography，不要在 `TextBlock`、`Button`、`TextBox`、`MenuItem` 上逐个重复 setter。统一放在 `Typography.axaml`，通过 `TextElement.FontFamily` 和 `TextElement.FontSize` 设置在窗口根部，让子级文本继承：

```xml
<Style Selector="Window">
    <Setter Property="TextElement.FontFamily" Value="{DynamicResource EditorFontFamilyGlobal}" />
    <Setter Property="TextElement.FontSize" Value="{DynamicResource EditorFontSizeDefault}" />
</Style>
```

只有需要局部变化时，才在具体 view 或 class 样式里覆盖 `FontSize`，例如标题、小字标签、表格辅助信息。

示例：

```xml
<Styles xmlns="https://github.com/avaloniaui">
    <Design.PreviewWith>
        <Border Background="{DynamicResource EditorBrushBase00}"
                Padding="16">
            <TextBox Text="Editable text" />
        </Border>
    </Design.PreviewWith>

    <Style Selector="Window">
        <Setter Property="TextElement.FontFamily" Value="{DynamicResource EditorFontFamilyGlobal}" />
        <Setter Property="TextElement.FontSize" Value="{DynamicResource EditorFontSizeDefault}" />
    </Style>
</Styles>
```

## 3. 局部覆盖

Shell 或 Feature 层只写带明确 class 的局部样式：

```xml
<Style Selector="Menu.editor-main-menu">
    <Setter Property="Height" Value="{DynamicResource EditorMainMenuHeight}" />
    <Setter Property="Background" Value="{DynamicResource EditorBrushBase01}" />
</Style>
```

不要在 Shell/Feature 里写裸选择器：

```xml
<!-- 不要这样做 -->
<Style Selector="TextBox">
</Style>
```

裸选择器会影响整个应用，必须放到 `UI/Controls/Base/Native`。

## 4. 什么时候写 ControlTheme

可以写 `ControlTheme` 的情况：

- 项目自定义控件，例如 `IconButton`。
- 需要完整替换某个控件的视觉树，并愿意维护 hover、pressed、disabled、focus、selected、keyboard、accessibility 等状态。
- 某个控件局部实例需要完全不同外观，可以用具名 theme 并显式赋给 `Theme`。

不要写全局内置控件 `ControlTheme` 的情况：

- 只是改颜色、字号、padding、背景、边框。
- 只是修菜单、输入框、列表项在某个面板里的视觉细节。
- 只是想覆盖 Fluent 的默认样式，但不想承担完整模板维护成本。

项目专用控件主题示例：

```xml
<ControlTheme x:Key="{x:Type base:IconButton}"
              TargetType="base:IconButton">
    <Setter Property="Background" Value="Transparent" />
    <Setter Property="BorderThickness" Value="0" />

    <Style Selector="^:pointerover">
        <Setter Property="Foreground" Value="{DynamicResource EditorBrushTextPrimary}" />
    </Style>
</ControlTheme>
```

`^` 表示当前 `ControlTheme` 的目标控件，适合在控件自己的 theme 内描述状态。

## 5. 控件类型选择

优先使用 Avalonia 原生控件行为：

- 普通按钮：`Button`
- 图标按钮：继承 `Button` 的 `IconButton`
- 输入框：`TextBox`
- 搜索框组合：`SearchBox` 可以是 `UserControl`
- 菜单：`Menu`、`MenuItem`、`Separator`
- 列表/层级树：`ListBox` 或 `ItemsControl`，大集合必须考虑虚拟化
- 分割条：`GridSplitter`

只有没有合适原生控件时，才创建新的 `TemplatedControl` 或 `Control + Render`。

## 6. 推荐结构

```text
UI/Styles/Tokens
  DeepDarkColors.axaml
  EditorMetrics.axaml

UI/Controls/Base
  IconButton.axaml
  IconButton.axaml.cs
  SearchBox.axaml
  SearchBox.axaml.cs

UI/Controls/Base/Native
  Index.axaml
  Typography.axaml
  TextBox.axaml

UI/Styles/Shell
  MainMenu.axaml
```

不要恢复聚合式 `EditorBaseControls.axaml`。该文件容易把“官方控件模板替换”和“项目样式覆盖”混在一起，后续定位问题困难。

## 7. 设计预览

资源字典或样式文件必须尽量提供 `Design.PreviewWith`：

```xml
<Styles xmlns="https://github.com/avaloniaui">
    <Design.PreviewWith>
        <Border Padding="16"
                Background="{DynamicResource EditorBrushBase00}">
            <Button Content="Button" />
        </Border>
    </Design.PreviewWith>
</Styles>
```

普通 `UserControl` 或 View 使用设计尺寸：

```xml
<UserControl xmlns="https://github.com/avaloniaui"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
             xmlns:d="http://schemas.microsoft.com/expression/blend/2008"
             xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"
             mc:Ignorable="d"
             d:DesignWidth="260"
             d:DesignHeight="24">
</UserControl>
```

设计时数据必须只服务 preview，不参与 runtime 服务注册、保存、命令执行或真实选择状态。

## 8. 审查清单

修改控件或样式时至少检查：

1. 是否先使用官方 Fluent theme，而不是重写内置控件完整模板。
2. 裸选择器是否只出现在 `UI/Controls/Base/Native`。
3. 原生控件全局覆盖是否按控件拆分，并由 `Native/Index.axaml` 聚合。
4. 每个全局覆盖文件是否有 `Design.PreviewWith`。
5. 默认字体是否通过 `Typography.axaml` 的 `TextElement.*` 继承属性设置，而不是每个控件重复 setter。
6. 是否只通过公开属性覆盖样式，避免依赖 Fluent 内部模板部件名。
7. 项目自定义控件是否保留独立 `ControlTheme` 和 `Design.PreviewWith`。
8. 大列表视觉树是否保持轻量，避免在行模板里堆复杂控件。
9. 是否运行 `dotnet test Editor.sln`、`git diff --check` 和编码检查。
